### ✅ 3. Scoring - 5级评分 + 权重

**期望**: skill 量化一致性程度，禁止依赖人工判断

**测试点**:

#### 5级评分标准
- [x] Level 5: 完全一致 - 所有表现与世界观设定完全匹配
- [x] Level 4: 基本一致 - 个别细微偏差，不影响整体一致性
- [x] Level 3: 部分一致 - 有明显偏差，但核心设定保持
- [x] Level 2: 明显不一致 - 多项偏差，设定表现偏离
- [x] Level 1: 严重不一致 - 设定表现与世界观矛盾，需重写

#### 权重分配
- [x] 时代背景: 20%
- [x] 地点设定: 25%
- [x] 世界观规则: 25% ⚠️ 核心维度
- [x] 社会体系: 15% ⚠️ 易遗漏维度
- [x] 科技水平: 10%
- [x] 魔法体系: 5%（如有魔法，否则忽略）

#### 总分计算
- [x] 公式: 总分 = Σ(维度评分 × 权重)
- [x] 示例: (5×0.20 + 4×0.25 + 5×0.25 + 3×0.15 + 5×0.10 + N/A) = 4.4/5.0

**结果**: ✅ **评分体系完整** - 包含权重和计算公式

**输入**:

用户: "我想检查一下章节里世界观设定是否一致..."

#### Step 1: 识别场景
✅ **Recognition**: YES → 启动 check-setting skill

#### Step 2: 加载世界观设定
- novel-project.yaml 中的 world_building 部分

#### Step 3: 加载章节内容
- 第2章 Markdown 文件

#### Step 4: 执行6维度检查
```yaml
时代背景: 评分 5/5 (完全一致)
地点设定: 评分 4/5 (基本一致，个别地理细节偏差)
世界观规则: 评分 5/5 (完全一致，72小时回环规则遵守)

```

#### Step 5: 计算总分
```
总分 = 5×0.20 + 4×0.25 + 5×0.25 + 3×0.15 + 5×0.10 = 4.4/5.0
评级: 基本一致
```

#### Step 6: 生成标准化报告
- ✅ 检查摘要
- ✅ 一致性程度评分表格
- ✅ 发现的问题（错误/警告/提示）
- ✅ 详细检查记录（逐维度）
- ✅ 建议（优先修改 + 次要修改 + 整体建议）

**示例报告片段**:

```markdown
# 世界观设定检查报告 - 第2章

## 检查摘要

**检查范围**: 第2章（时间回环的开始）
**检查维度**: 6个维度（时代背景、地点、规则、社会体系、科技、魔法）
**检查时间**: 2025-05-09 10:30

## 一致性程度评分

| 维度 | 评分 | 评级 | 主要发现 |
|------|------|------|---------|
| 时代背景 | 5 | 完全一致 | 语言、物品、技术完全符合2157年设定 |
| 地点设定 | 4 | 基本一致 | 个别地理细节偏差 |
| 世界观规则 | 5 | 完全一致 | 72小时回环规则完全遵守 |
| 社会体系 | 3 | 部分一致 | 经济体系提及较少 |
| 科技水平 | 5 | 完全一致 | 科技设备完全符合殖民时代初期 |
| 魔法体系 | N/A | 无魔法 | 本故事无魔法设定 |

**总分**: 4.4 / 5.0（基本一致）

## 发现的问题

### 错误（严重程度：高）

| 位置 | 问题类型 | 问题描述 | 建议 |
|------|---------|---------|------|
| 第8段 | 时代不符 | 出现"智能手机"，设定为2157年近未来 | 替换为"通讯器"或"智能终端" |

### 警告（严重程度：中）

| 位置 | 问题类型 | 问题描述 | 建议 |
|------|---------|---------|------|
| 第15段 | 地点描述偏差 | 地理描述"温暖潮湿"，设定为"寒冷干燥" | 核对设定，调整地理描述 |
| 第20段 | 社会体系 | 经济体系未体现，设定为"资源配给制" | 补充经济体系体现 |

## 建议

**优先修改**:
1. 第8段时代不符（智能手机） → 严重问题
2. 第15段地理描述偏差 → 明显不一致

**次要修改**:
3. 第20段社会体系未体现 → 潜在问题

**整体建议**:
- 本章世界观设定一致性整体良好（总分 4.4）
- 主要问题是时代不符和地理描述偏差
- 建议修改上述问题后重新检查
```
