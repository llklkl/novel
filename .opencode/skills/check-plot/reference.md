## 评分标准详细定义

```yaml
logic_score:
  - level: 5
    label: "完全合理"
    criteria: "所有情节逻辑完全合理，无矛盾"

  - level: 4
    label: "基本合理"
    criteria: "个别因果关系弱，不影响整体"

  - level: 3
    label: "部分合理"
    criteria: "有逻辑问题，但核心情节保持"

  - level: 2
    label: "明显不合理"
    criteria: "多项逻辑问题"

  - level: 1
    label: "严重不合理"
    criteria: "情节逻辑矛盾，需重写"
```

**权重分配：**
- 因果关系：权重 30%
- 情节线完整性：权重 25%
- 人物动机：权重 20%
- 情节漏洞：权重 15%
- 伏笔揭示：权重 10%

## 各维度详细检查方法

### 1. 因果关系合理性检查

**检查情节发展是否有合理的因果关系：**

**检查方法：**
1. 提取章节中的事件列表（按出现顺序）
2. 分析每个事件之间的因果关系
3. 检查因果关系是否合理（无断裂）

**不一致识别标准：**
- **明显不合理**: 因果关系断裂
  - 例：事件B发生，但无事件A导致
- **微妙不合理**: 因果关系弱
  - 例：因果关系不明确或不充分
- **潜在问题**: 因果关系可能缺失

**评分标准：**
- 5分：因果关系完全合理，清晰明确
- 4分：个别因果关系弱
- 3分：有因果关系问题但核心保持
- 2分：多项因果关系问题
- 1分：因果关系严重断裂

### 2. 情节线完整性检查

**检查情节线是否有开头、发展、结尾：**

**检查方法：**
1. 提取大纲中的情节线设定
2. 识别章节中的情节线发展
3. 检查情节线是否完整（无遗漏）

**不一致识别标准：**
- **明显不合理**: 情节线不完整
  - 例：情节线有开头但无结尾
- **微妙不合理**: 情节线发展不足
  - 例：情节线发展过快或过慢
- **潜在问题**: 情节线可能遗漏

**评分标准：**
- 5分：情节线完全完整
- 4分：个别情节线发展不足
- 3分：有情节线问题但核心保持
- 2分：多项情节线问题
- 1分：情节线严重不完整

### 3. 人物动机充分性检查

**检查人物行为是否符合动机设定：**

**检查方法：**
1. 提取角色档案中的 motivation 信息
2. 识别章节中人物的关键行为和决策
3. 检查行为是否有充分动机

**不一致识别标准：**
- **明显不合理**: 动机不充分
  - 例：角色做出关键决策但无动机
- **微妙不合理**: 动机与档案不符
  - 例：行为与角色档案中的动机矛盾
- **潜在问题**: 动机可能缺失

**评分标准：**
- 5分：所有动机充分且符合设定
- 4分：个别动机不充分
- 3分：有动机问题但核心保持
- 2分：多项动机问题
- 1分：动机严重缺失或不符

### 4. 情节漏洞检查

**检查是否有情节漏洞或矛盾：**

**检查方法：**
1. 识别章节中的情节跳跃
2. 检查是否有未解释的跳跃（情节漏洞）
3. 检查是否有情节矛盾

**不一致识别标准：**
- **明显不合理**: 情节漏洞
  - 例：角色突然出现在某地，但前文未交代其如何到达
- **微妙不合理**: 信息缺失
  - 例：关键信息未交代（如角色如何获得某物品）
- **潜在问题**: 情节跳跃可能需要补充

**评分标准：**
- 5分：无情节漏洞
- 4分：个别信息缺失
- 3分：有情节漏洞但核心保持
- 2分：多项情节漏洞
- 1分：严重情节漏洞

### 5. 伏笔揭示检查

**检查本章是否揭示大纲中的伏笔：**

**检查方法：**
1. 提取大纲中的 foreshadowing 设定
2. 检查本章是否揭示计划中的伏笔
3. 检查伏笔揭示是否合理

**不一致识别标准：**
- **明显不合理**: 伏笔遗漏
  - 例：大纲中伏笔应在本章揭示，但未揭示
- **微妙不合理**: 伏笔揭示不合理
  - 例：伏笔揭示与真相不符
- **潜在问题**: 伏笔揭示可能不足

**评分标准：**
- 5分：所有伏笔按计划揭示
- 4分：个别伏笔未揭示
- 3分：有伏笔问题但核心保持
- 2分：多项伏笔问题
- 1分：伏笔严重遗漏或不符
