# 情节逻辑检查Skill

## Overview
检查章节内容中情节逻辑的合理性，包括因果关系、情节线完整性、人物动机、情节漏洞和伏笔揭示，生成标准化的检查报告。

**核心原则: 情节逻辑检查 = 标准化检查清单 + 系统化检查流程 + 标准化报告格式 + 合理性程度量化。**

手工检查方法会识别情节发展并人工检查逻辑，但缺乏标准化流程，无法量化合理性程度，没有标准化报告格式，对情节漏洞和伏笔不敏感，每次检查可能不一致。系统化方法确保完整性和可重复性。

## Pattern Recognition - 何时使用此skill

**使用此skill的场景**：
- 用户说"我想检查一下章节里情节逻辑是否合理..." → **启动情节逻辑检查**
- 用户说"我想检查因果关系、情节漏洞是否有问题" → **启动情节逻辑检查**
- 用户说"我想检查人物动机是否充分" → **启动情节逻辑检查**
- 用户说"我完成了章节撰写，需要做什么检查？" → **建议使用此skill（以及其他 check-* skills）**

**Red Flags - 必须使用此skill**：
- 尝试手工检查，没有预定义检查清单（禁止）
- 尝试依赖经验判断"合理性程度"，无法量化（禁止）
- 尝试没有标准化报告格式（禁止）
- 尝试对情节漏洞和伏笔不敏感（禁止）
- 尝试每次检查不一致（禁止）

**所有这些意味着：用户需要系统化的情节逻辑检查过程，必须使用此skill。**

## 工作流程

### 1. 加载情节信息
- 读取 novel-project.yaml 中的 outline.chapters 部分
- 提取每个章节的情节设定（plot_points, conflicts, foreshadowing）
- 读取角色档案中的动机信息
- **完成标准**: 成功加载情节相关信息

### 2. 加载章节内容
- 读取指定章节的 Markdown 文件
- 标记每个情节节点（事件、冲突、伏笔）
- **完成标准**: 章节内容加载成功，情节节点已标记

### 3. 初始化检查清单（强制使用标准化清单）

**禁止手工检查！必须使用以下检查清单：**

```yaml
check_list:
  plot_dimensions:
    - dimension: "因果关系合理性"
      check_items:
        - "情节发展是否有合理的因果关系"
        - "事件A是否合理导致事件B"
        - "因果关系是否自洽"
        - "是否有因果关系断裂"
      severity_threshold: "因果关系不合理"

    - dimension: "情节线完整性"
      check_items:
        - "情节线是否有开头、发展、结尾"
        - "是否存在未解决的情节线"
        - "情节线是否与大纲一致"
        - "是否有情节线遗漏"
      severity_threshold: "情节线不完整"

    - dimension: "人物动机充分性"
      check_items:
        - "人物行为是否符合动机设定"
        - "关键决策是否有充分动机"
        - "动机是否与角色档案一致"
        - "是否有动机缺失"
      severity_threshold: "人物动机不充分"

    - dimension: "情节漏洞检查"
      check_items:
        - "是否有情节漏洞（未解释的跳跃）"
        - "是否有情节矛盾"
        - "是否有逻辑矛盾"
        - "是否有信息缺失"
      severity_threshold: "情节漏洞"

    - dimension: "伏笔揭示检查"
      check_items:
        - "本章是否揭示大纲中的伏笔"
        - "伏笔揭示是否合理"
        - "是否有伏笔遗漏"
        - "伏笔是否与真相对应"
      severity_threshold: "伏笔揭示问题"
```

**完成标准**: 初始化完整的检查清单（5个维度）

### 4. 逐维度执行检查（系统化流程）

**检查方法：**

**Step 1: 识别情节节点**
- 扫描章节内容，标记所有情节节点（事件、冲突、伏笔）
- 分类标记：因果关系、情节线、人物动机、情节漏洞、伏笔揭示

**Step 2: 对比大纲情节设定**
- 将章节中的情节表现与大纲中的 plot_points, conflicts, foreshadowing 对比
- 检查情节发展是否与大纲一致
- 检查伏笔是否按计划揭示

**Step 3: 识别不一致类型**
- **明显不合理**: 情节逻辑直接矛盾
- **微妙不合理**: 因果关系弱或动机不足
- **潜在问题**: 情节线不完整或伏笔遗漏

**Step 4: 量化合理性程度**

```yaml
logic_score:
  - level: 5
    label: "完全合理"
    criteria: "所有情节逻辑完全合理，无矛盾"

  - level: 4
    label: "基本合理"
    criteria: "个别因果关系弱，不影响整体"

  - level: 3
    label: "部分合理"
    criteria: "有逻辑问题，但核心情节保持"

  - level: 2
    label: "明显不合理"
    criteria: "多项逻辑问题"

  - level: 1
    label: "严重不合理"
    criteria: "情节逻辑矛盾，需重写"
```

**每个维度评分后计算总分：**
- 因果关系：权重 30%
- 情节线完整性：权重 25%
- 人物动机：权重 20%
- 情节漏洞：权重 15%
- 伏笔揭示：权重 10%

**完成标准**: 每个维度的合理性程度已量化（1-5分）

### 5. 因果关系合理性检查（核心）

**检查情节发展是否有合理的因果关系：**

**检查方法：**
1. 提取章节中的事件列表（按出现顺序）
2. 分析每个事件之间的因果关系
3. 检查因果关系是否合理（无断裂）

**不一致识别标准：**
- **明显不合理**: 因果关系断裂
  - 例：事件B发生，但无事件A导致
- **微妙不合理**: 因果关系弱
  - 例：因果关系不明确或不充分
- **潜在问题**: 因果关系可能缺失

**评分标准：**
- 5分：因果关系完全合理，清晰明确
- 4分：个别因果关系弱
- 3分：有因果关系问题但核心保持
- 2分：多项因果关系问题
- 1分：因果关系严重断裂

### 6. 情节线完整性检查（易遗漏）

**检查情节线是否有开头、发展、结尾：**

**检查方法：**
1. 提取大纲中的情节线设定
2. 识别章节中的情节线发展
3. 检查情节线是否完整（无遗漏）

**不一致识别标准：**
- **明显不合理**: 情节线不完整
  - 例：情节线有开头但无结尾
- **微妙不合理**: 情节线发展不足
  - 例：情节线发展过快或过慢
- **潜在问题**: 情节线可能遗漏

**评分标准：**
- 5分：情节线完全完整
- 4分：个别情节线发展不足
- 3分：有情节线问题但核心保持
- 2分：多项情节线问题
- 1分：情节线严重不完整

### 7. 人物动机充分性检查

**检查人物行为是否符合动机设定：**

**检查方法：**
1. 提取角色档案中的 motivation 信息
2. 识别章节中人物的关键行为和决策
3. 检查行为是否有充分动机

**不一致识别标准：**
- **明显不合理**: 动机不充分
  - 例：角色做出关键决策但无动机
- **微妙不合理**: 动机与档案不符
  - 例：行为与角色档案中的动机矛盾
- **潜在问题**: 动机可能缺失

**评分标准：**
- 5分：所有动机充分且符合设定
- 4分：个别动机不充分
- 3分：有动机问题但核心保持
- 2分：多项动机问题
- 1分：动机严重缺失或不符

### 8. 情节漏洞检查（核心）

**检查是否有情节漏洞或矛盾：**

**检查方法：**
1. 识别章节中的情节跳跃
2. 检查是否有未解释的跳跃（情节漏洞）
3. 检查是否有情节矛盾

**不一致识别标准：**
- **明显不合理**: 情节漏洞
  - 例：角色突然出现在某地，但前文未交代其如何到达
- **微妙不合理**: 信息缺失
  - 例：关键信息未交代（如角色如何获得某物品）
- **潜在问题**: 情节跳跃可能需要补充

**评分标准：**
- 5分：无情节漏洞
- 4分：个别信息缺失
- 3分：有情节漏洞但核心保持
- 2分：多项情节漏洞
- 1分：严重情节漏洞

### 9. 伏笔揭示检查（易遗漏）

**检查本章是否揭示大纲中的伏笔：**

**检查方法：**
1. 提取大纲中的 foreshadowing 设定
2. 检查本章是否揭示计划中的伏笔
3. 检查伏笔揭示是否合理

**不一致识别标准：**
- **明显不合理**: 伏笔遗漏
  - 例：大纲中伏笔应在本章揭示，但未揭示
- **微妙不合理**: 伏笔揭示不合理
  - 例：伏笔揭示与真相不符
- **潜在问题**: 伏笔揭示可能不足

**评分标准：**
- 5分：所有伏笔按计划揭示
- 4分：个别伏笔未揭示
- 3分：有伏笔问题但核心保持
- 2分：多项伏笔问题
- 1分：伏笔严重遗漏或不符

### 10. 生成标准化报告（强制格式）

**禁止没有标准化报告格式！必须使用以下格式：**

```markdown
# 情节逻辑检查报告 - 第X章

## 检查摘要

**检查范围**: 第X章（标题）
**检查维度**: 5个维度（因果关系、情节线、人物动机、情节漏洞、伏笔揭示）
**检查时间**: YYYY-MM-DD HH:MM

## 合理性程度评分

| 维度 | 评分 | 评级 | 主要发现 |
|------|------|------|---------|
| 因果关系 | 5 | 完全合理 | 因果关系清晰明确 |
| 情节线完整性 | 4 | 基本合理 | 个别情节线发展不足 |
| 人物动机 | 3 | 部分合理 | 个别动机不充分 |
| 情节漏洞 | 2 | 明显问题 | 有情节漏洞 |
| 伏笔揭示 | 5 | 完全合理 | 伏笔按计划揭示 |

**总分**: 3.85 / 5.0（部分合理）

**评分标准**: 5分=完全合理, 4分=基本合理, 3分=部分合理, 2分=明显问题, 1分=严重不合理

## 发现的问题

### 错误（严重程度：高）

| 位置 | 问题类型 | 问题描述 | 建议 |
|------|---------|---------|------|
| 第6段 | 情节漏洞 | 角色突然出现但前文未交代其如何到达 | 补充角色到达说明 |

### 警告（严重程度：中）

| 位置 | 问题类型 | 问题描述 | 建议 |
|------|---------|---------|------|
| 第12段 | 人物动机 | 角色做出关键决策但动机不充分 | 增加动机触发事件 |
| 第15段 | 情节线 | 情节线发展过快，缺少中间过程 | 补充中间过程段落 |

### 提示（严重程度：低）

| 位置 | 问题类型 | 问题描述 | 建议 |
|------|---------|---------|------|
| 第20段 | 情节线 | 情节线结尾不够完整 | 补充情节线结尾 |

## 详细检查记录

### 因果关系合理性检查（评分：5/5）

**事件因果链**:
1. 收到消息 → 决定行动 ✓
2. 决定行动 → 出发 ✓
3. 出发 → 到达 ✓
4. 到达 → 发现真相 ✓

**问题列表**: 无问题发现

### 情节漏洞检查（评分：2/5）

**情节跳跃列表**:
- 第6段：角色突然出现在目的地 → 前文未交代其如何到达 ⚠️（严重）
- 第10段：角色获得关键物品 → 未交代其如何获得 ⚠️（明显）

**问题列表**:
- 第6段：情节漏洞（角色突然出现） → 建议：补充到达过程说明
- 第10段：信息缺失（物品来源） → 建议：补充物品来源说明

...

## 建议

**优先修改**:
1. 第6段情节漏洞（角色突然出现） → 严重问题
2. 第10段信息缺失（物品来源） → 明显问题

**次要修改**:
3. 第12段人物动机不充分 → 潜在问题
4. 第15段情节线发展过快 → 微妙问题

**整体建议**:
- 本章情节逻辑部分合理（总分 3.85）
- 主要问题是情节漏洞和人物动机
- 建议修改上述问题后重新检查
```

### 11. 输出结果
- 生成标准化检查报告
- 按严重程度排序（错误 > 警告 > 提示）
- 提供具体修改建议
- **完成标准**: 检查报告已生成，包含评分和问题列表

## 禁止行为

**以下行为被明确禁止：**

1. **禁止手工检查**
   - 不允许没有预定义检查清单
   - 必须使用标准化检查清单（5个维度）

2. **禁止无法量化合理性程度**
   - 不允许依赖经验判断"合理性程度"
   - 必须使用评分标准（1-5分）量化

3. **禁止没有标准化报告格式**
   - 不允许随意报告格式
   - 必须使用标准化报告格式（摘要、评分、问题列表、详细记录、建议）

4. **禁止遗漏关键检查项**
   - 不允许不检查情节线完整性（易遗漏）
   - 不允许不检查伏笔揭示（易遗漏）

5. **禁止检查不一致**
   - 不允许每次检查方法不一致
   - 必须使用系统化检查流程

| 无法量化合理性程度 | 判断主观，无法衡量 | 强制使用评分标准（1-5分）量化 |
| 没有标准化报告格式 | 报告随意，难以使用 | 强制使用标准化报告格式 |
| 对情节漏洞和伏笔不敏感 | 遗漏问题 | 明确易遗漏项（情节线、伏笔） |
| 每次检查不一致 | 可重复性低 | 系统化方法确保可重复性 |

1. 因果关系（权重30%）⚠️ 核心
2. 情节线完整性（权重25%）⚠️ 易遗漏
3. 人物动机（权重20%）
4. 情节漏洞（权重15%）⚠️ 核心
5. 伏笔揭示（权重10%）⚠️ 易遗漏

**评分标准（5级）**：
- 5分：完全合理
- 4分：基本合理（个别因果关系弱）
- 3分：部分合理（有逻辑问题）
- 2分：明显不合理（多项问题）
- 1分：严重不合理（矛盾）

**不一致类型（3种）**：
- 明显不合理：情节逻辑直接矛盾
- 微妙不合理：因果关系弱或动机不足
- 潜在问题：情节线不完整或伏笔遗漏

**关键检查项（易遗漏）**：
- ⚠️ 情节线完整性
- ⚠️ 伏笔揭示

**报告格式（5部分）**：
1. 检查摘要
2. 合理性程度评分（表格）
3. 发现的问题（错误/警告/提示）
4. 详细检查记录（逐维度）

## AI角色
检查助手模式 - 系统化检查、量化评分、标准化报告

## 注意事项
- 检查要系统化，不能手工
- 评分要量化，不能依赖经验判断
- 报告要标准化，便于用户理解和修改
- 易遗漏的检查项要特别关注（情节线、伏笔）
- 对情节漏洞要敏感

## 错误处理

- **配置文件不存在**: 提示用户先运行 novel-project skill 创建项目
- **无大纲信息**: 提示用户先完成 outline-design 阶段
- **章节内容为空**: 提示用户先完成章节撰写
- **无情节设定**: 提示用户补充大纲中的 plot_points, foreshadowing
